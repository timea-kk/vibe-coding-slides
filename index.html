how I worked with AI

A structured agent setup

Agent selector agents/agent_selection.md
Automatically detects which agent applies based on the task — no manual selection needed. Multiple agents can apply simultaneously. Claude figures it out silently.
Writer agent agents/writer.yml
Handles all copy, case study content, and narrative sections. Knows Timea's voice, tone rules, and writing conventions — including a hard ban on em dashes.
Developer agent agents/developer.yml
Handles Vue components, styling, animations, and testing. Knows the full stack, file structure, breakpoint rules, and when to propose a shared component vs. a one-off.
Design system agent agents/design-system.yml
Handles design tokens, component APIs, and documentation. Enforces that no token or component is duplicated or overridden without explicit instruction.

Project 01 — Portfolio
github actions

My first CI/CD pipeline

Job 1 — runs simultaneously with jobs 2 & 3
✦ Lint
Checks that code is written consistently and flags bad patterns — like a spell-checker for code. Catches unused variables or inconsistent formatting before they become bugs.
Job 2 — runs simultaneously with jobs 1 & 3
✦ Type check
TypeScript verifies every variable and function is used correctly. Catches mistakes — like passing the wrong kind of data to a component — before the browser ever runs the code.
Job 3 — runs simultaneously with jobs 1 & 2
✦ Test + coverage
Runs all 291 automated tests. If a change accidentally broke something elsewhere, this is where it shows up. Coverage is reported to Codecov to track over time.
↓   all three must pass before job 4 runs
Job 4
🔨 Build
Compiles all source files into the final HTML, CSS, and JavaScript a browser can read. Like exporting a final PDF from Figma — confirms the whole thing assembles cleanly with no errors.
↓   only runs on pushes to main
Job 5
🚀 Deploy
Pushes the built site live to Vercel. Vercel's own auto-deploy is disabled — this pipeline job is the only thing that can publish to production, so nothing goes live unless all four previous jobs passed.

What it does

Project 02 — Chrome Extension
github actions

CI/CD for a Chrome extension

Job 1 — runs simultaneously with job 2
✦ Lint
Checks code style and catches bad patterns. Same as the portfolio pipeline — catches problems before they become bugs.
Job 2 — runs simultaneously with job 1
✦ Test + coverage
Runs the automated test suite and uploads coverage to Codecov. No type check job — TypeScript is compiled separately as part of the build step.
↓   both must pass before job 3 runs
Job 3
📦 Package extension
Bundles all the extension files (manifest, HTML, JS, icons, images) into a ZIP file. This is the actual file that gets submitted to the Chrome Web Store — like creating the final deliverable.
↓   only runs on pushes to main
Job 4
🚀 Publish to Chrome Web Store
Pushes the ZIP live to the Chrome Web Store via Google OAuth. Requires 4 secrets set in GitHub: extension ID, Google client ID, client secret, and refresh token.

Steps to go from fork to published extension
01
Read the source files
Every .ts file has plain English comments written for designers. Read top to bottom before changing anything.
02
Set up your photo library
Tell Claude Code your Unsplash API key and theme. It downloads 50 photos per time period and fills in location and photographer credits.
03
Rename the extension
One prompt updates manifest.json, the page title, settings page, and every reference to "Arctic Scapes" in the codebase.
04
Rewrite the greetings
Describe your theme and tone. Claude Code rewrites all three time-of-day pools and the 20-line surprise pool.
05
Run a local preview
Ask Claude Code to start it. Opens at localhost:3000 — review photos and layout before loading into Chrome.
06
Load into Chrome
Run npm run build, go to chrome://extensions, enable Developer mode, click Load unpacked.
07
Connect Codecov
Ask Claude Code to set up the GitHub Actions workflow that reports test coverage on every push.
08
Publish to Chrome Web Store
One-time $5 Google developer fee. Claude Code creates the ZIP and walks you through the dashboard.
